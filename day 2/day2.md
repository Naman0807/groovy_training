## 1. Live Demo — Continue.dev / Cline Shortcuts

Krunal ran a 20-min live demo showing the AI-IDE workflow used at Groovy Web.

### Key Shortcuts Learned

| Shortcut | Action | Notes |
| :------- | :----- | :---- |
| **`Ctrl+L`** | Open Continue.dev chat panel | Main prompt interface — used 90% of the time |
| **`Ctrl+I`** | Inline code edit / generate | Highlight code → `Ctrl+I` → prompt → replaces selection |
| **`Ctrl+Shift+L`** | Add selected code to chat context | Quick way to reference a specific function |
| **`@file`** | Attach entire file as context | `@file server.js` — attaches the full file content |
| **`@folder`** | Attach folder structure as context | `@folder backend/` — adds all files in that folder |
| **`@codebase`** | Attach entire project context | Best for cross-file refactors — uses embedding search |
| **`@web`** | Search the web for latest docs | Useful for library versions, API changes |
| **`Alt+/`** | Quick inline completion (Codeium) | Tab autocomplete for single lines |

### Demo Project

Krunal live-prompted a **CLI CSV parser** in 4 minutes:

```text
@codebase Create a Node.js CLI tool that reads a CSV file passed as argv[2],
parses it, and outputs formatted JSON to stdout. Handle quoted fields and
newlines inside fields. Use only built-in Node modules.
```

> **Time:** Prompt → working code in under 3 minutes. No editing needed.
>
> **Nauman's comment:** "See how `@codebase` with the right framing replaces three rounds of back-and-forth? That's the skill."

## 2. Context Attaching Practice

After the demo, I practiced the four context modes on the starter project.

### Exercise Log

| Mode | What I Attached | Prompt | AI Response Quality |
| :--- | :-------------- | :----- | :------------------ |
| `@file` | `package.json` | "What scripts are defined here?" | Correct — listed all 4 scripts verbatim |
| `@folder` | `backend/` | "Summarize the folder structure" | Listed files correctly but hallucinated a `config/` dir that doesn't exist |
| `@codebase` | Entire project | "Where are API routes defined?" | Found `server.js` and listed all 4 routes accurately |
| `@web` | — | "Latest version of Express.js" | Returned v4.19.2 with changelog link — correct |

### Context Mistake Log

1. **`@folder` hallucination:** When I attached `@folder backend/`, the AI claimed there was a `config/database.js` file. I checked — it doesn't exist.
   - **Lesson:** AI infers missing structure. Always verify with actual file tree after `@folder`.
2. **`@codebase` timeout:** First `@codebase` scan took ~20 seconds on the starter project (which is tiny). For larger repos, this could be slow.
   - **Workaround:** Use `@file` + `@folder` for targeted context instead.

## 6. Code Review with Krunal

### Session Notes

Krunal pulled up my repo and did a 15-minute live review over a Zoom call.

#### What was checked

- [x] All 4 CRUD endpoints working? — Yes
- [x] Input validation? — Title validation on POST, missing on PUT
- [x] Error handling? — 404 for missing IDs, 400 for empty title. Missing 500 handler
- [x] CORS configured? — Yes, wildcard `*`
- [x] Frontend error handling? — `.catch(console.error)` on all fetches — minimal, works for now
- [x] No secrets committed? — `.env` in `.gitignore`, clean
- [x] README complete? — Yes, prompt table is excellent

#### Issues Found

1. **PUT endpoint missing title validation vs. POST** — POST checks empty title but PUT doesn't (it only checks if `title` is provided, not if it's empty string). AI inconsistency.
2. **No error state in UI** — If the backend is down, the app silently fails (just console.error). No visual feedback.
3. **`proxy` field unused** — Hardcoded `http://localhost:3001` in fetch URLs means the CRA proxy config does nothing. Either remove proxy or use relative URLs.

### ✅ 1 Thing AI Did Well

> "The POST validation — checking for whitespace-only titles — was handled without being prompted. The AI anticipated a real edge case that many junior devs miss on their first CRUD API."

### ❌ 1 Thing AI Failed At

> "The `proxy` field in `frontend/package.json` is set but the frontend still uses a hardcoded `http://localhost:3001` URL in `App.js`. This means the proxy isn't actually being used. In development, you should either remove the hardcoded URL and rely on the proxy, or remove the proxy config. This is a common inconsistency that AI tools generate — it works, but it's not clean."
