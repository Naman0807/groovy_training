## 📋 Overview

This repository documents a 14-day intensive training program on AI-First Engineering — learning to build production-quality software by writing prompts instead of typing code.

The training covers:
- **AI-IDE integration** (Continue.dev, Cline, Codeium)
- **Prompt engineering** (context attaching, role-based prompting, multi-step workflows)
- **API integration** (Google Gemini API, SDKs, function calling)
- **Mini-projects** (TODO App, Smart Doc Q&A, RAG pipeline)
- **Agent frameworks** (LangChain, LlamaIndex, Pure SDK agents)
- **Multi-agent systems** (supervisor-worker patterns)

## 🎯 Key Learnings

1. **Prompts > Keystrokes** — The bottleneck shifted from writing code to describing problems precisely.
2. **Context is king** — `@file`, `@folder`, `@codebase` attachments eliminate back-and-forth.
3. **Review AI output like a junior's PR** — Hidden inconsistencies (proxy config, missing validation) are common.
4. **Output format specification is critical** — JSON format instructions reduced parsing errors from 40% to 5%.
5. **Build raw first, then framework** — Building the pure SDK agent revealed what LangChain and LlamaIndex do under the hood.
6. **Memory is the hardest part of agent design** — Short-term is trivial; long-term summarization trades detail for context.
